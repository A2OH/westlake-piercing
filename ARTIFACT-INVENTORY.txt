NOTE on md5 reconciliation: the SAME BCP jar shows competing md5s across sessions
per feature (e.g. ohaf 300581d1->4690cae1; framework.jar f5fd86ef->e6f9e1a3; arb
d5d39a05->6e32a253->c026e80c; libart 2813065e->ba40f173; boot-framework.oat
54001902->290e4499). The values above are the LIVE demo-ready set (verified on
device 2026-06-25). A from-scratch build must apply ALL the smali/source patches
to each component and do ONE final boot regen — do not deploy per-session md5s in
isolation; regenerate the boot image to match whatever jar set you assemble.
