**2026-06-03: noice child now has AF_INET network (gid 3003).** Goal "run full featured UI with network supported API data".

## Root cause of the network EPERM
OHOS kernel has paranoid-network: AF_INET `socket()` needs the process in AID_INET (gid 3003). The adapter's `appspawn-x` **skips `OH_TLV_INTERNET_INFO`** ("not on hot path", spawn_server.cpp ~588) so app children launch with `gids_count=1` (only the app gid 3099, NO inet) → `socket()` → EPERM. SELinux is permissive, NOT the blocker. Device host (root) HAS internet (8.8.8.8 pings 17ms; api.trynoice.com resolves to 35.94.160.101 via resolv.conf 114.114.114.114/8.8.8.8 — AWS host blocks ICMP but HTTPS works).

## Fix: LD_PRELOAD setgroups interposer
`libsetgidhook.so` (source /tmp/setgidhook.c) intercepts `setgroups(n,list)` (called by appspawn-x child_main.cpp applyDac before uid drop) and appends AID_INET(3003)+AID_NET_RAW(3004). Built locally with OHOS clang for arm32:
`prebuilts/clang/ohos/linux-x86_64/llvm/bin/clang --target=arm-linux-ohos -fPIC -shared -O2 -isystem out/rk3568/obj/third_party/musl/usr/include/arm-linux-ohos -isystem .../include -nostdlib -o libsetgidhook.so setgidhook.c .../lib/arm-linux-ohos/libc.so`. Deployed to `/system/android/lib/libsetgidhook.so`. Logs each call to `/data/local/tmp/setgid.log`.
**PROVEN:** child pid=2179 `setgid.log`: `[setgidhook] pid=2179 in=1 out=3 (added inet=1 raw=1)`; `/proc/2179/status Groups: 3003 3004 3099`. noice renders the full sound library (声音库 LIFE: Birds/Crickets/Heartbeat/Purring Cat).

## CRITICAL: LD_PRELOAD must come via start_asx.sh, NOT the init cfg
appspawn_x.cfg has `setuid:true` + secon `u:r:appspawn:s0` → the `appspawn` SELinux domain sets AT_SECURE → the dynamic linker STRIPS LD_PRELOAD (env-array AND `sh -c export` in the cfg path BOTH fail; w14supp/setgidhook never load → noice fork-fails: AMS "app may fork fail or not running"). The shell domain is NOT AT_SECURE, so `start_asx.sh` (run from hdc shell) loads preloads fine. **So: do NOT put LD_PRELOAD in appspawn_x.cfg.** (I reverted the cfg path; init-managed appspawn-x is preload-less and useless for apps.)

## The working bring-up procedure (after every reboot)
appspawn-x is ondemand+transient (renames comm to `main`, exits when idle). To make a shell-launched one the live server:
1. `start_asx.sh` LD_PRELOAD = `libsetgidhook.so:libw14supp.so:libgaihook.so` (w14supp = substrate, REQUIRED; gaihook stubs getaddrinfo — DROP for real DNS).
2. `$HDC shell "setsid sh /data/local/tmp/start_asx.sh </dev/null >/dev/null 2>&1 &"` (setsid detaches; survives hdc exit; reparents to init ppid=1).
3. **MANDATORY (else AMS can't connect → no spawn request reaches it):** `chmod 0666 /dev/unix/socket/AppSpawnX; chcon u:object_r:appspawn_socket:s0 /dev/unix/socket/AppSpawnX; setenforce 0` (per HANDOFF-2026-05-28-END.md). Wrong socket SELinux label = silent no-route.
4. Verify: `grep -l setgidhook /proc/*/maps` finds ONE appspawn-x (comm=main, ppid=1, setgid=4 mappings). Two appspawn-x = socket conflict = all apps fail (THEME-FIX-HANDOFF).
5. Launch: `aa start -a com.github.ashutoshgngwr.noice.activity.MainActivity -b com.github.ashutoshgngwr.noice` (unlock keyguard first: mid-screen swipe `uinput -T -m 360 900 360 300`).

## 2026-06-03 NETWORK WORKS END-TO-END (app uid reaches real cdn.trynoice.com)
Full chain now functional. PROOF (`conntest` dropping to uid 13731): `socket(AF_INET,STREAM) rc=3`; `connect(35.94.160.101:443) rc=0`; `send rc=43`; `recv n=145` → `HTTP/1.1 400 Bad Request / Server: nginx` = **the real cdn.trynoice.com nginx responded** (400 only because plaintext sent to HTTPS port). The 4 layers fixed:
1. **inet gid** — setgidhook adds 3003/3004 + OHOS 1097(netsys_socket)/1098 (necessary, not sufficient alone).
2. **connectivity offline-gate** — noice's `ContextExtKt.getInternetConnectivityFlow$1` (ext/a.smali method a) callbackFlow reports offline (NetworkCallback never fires via the adapter ConnectivityManager). PATCHED in /tmp/noice_smali: forced initial emit to true (`const/4 v3,0x1` before Boolean.valueOf at the `:goto_5b`→`Lf8/q;->r` trySend). Now noice ATTEMPTS the fetch.
3. **DNS** — OHOS musl getaddrinfo routes through netsys per-netid; sandboxed child has no netid → EAI_NONAME/AGAIN. Fixed with `libdnshook.so` (getaddrinfo interposer doing direct UDP A-query to 8.8.8.8:53; child has inet so UDP works). Resolves cdn.trynoice.com→35.94.160.101.
4. **socket() EPERM — THE REAL GATE: OHOS netsys cgroup-eBPF per-UID internet permission.** Full caps + SELinux-permissive + no-seccomp + inet gids ALL present, yet socket(AF_INET) EPERM. Root cause: `/system/etc/bpf/netsys.o` program `cgroup_sock/inet_create_socket` reads map `oh_sock_permission_map` (id varies, key=u32 uid, val=1 byte): **val==0 → DENY, val!=0 or absent → ALLOW** (decoded from BPF disasm: lookup uid; if NULL→allow; if *val!=0→allow; else deny). noice's uid 13731 was in the map with val=**0** (BMS added it as no-internet since noice has empty OHOS permissions). FIX: `/data/local/tmp/bpfgrant 13731 oh_sock_permission_map` (custom tool, built w/ OHOS clang + linux/bpf.h, enumerates maps by BPF_MAP_GET_NEXT_ID, finds oh_sock_permission_map by 15-char-truncated name, sets [uid]=1). After grant: socket rc=61, dnshook resolves, TCP connect rc=0 to real CDN.
**Tools (all /data/local/tmp + /system/android/lib): libsetgidhook.so, libdnshook.so, libnetlog.so, bpfgrant, conntest, dnstest.** start_asx.sh LD_PRELOAD = setgidhook:w14supp:dnshook:netlog. **Grant must be (re)applied after netsys may reset it — re-grant in a loop during launch, or persist at bringup.** noice's own okhttp HTTPS fetch still gated only by the ~50% flaky initChild spawn (network itself proven working).
