## ArkUI ARM32 on QEMU
- [TLS fix details](project_arkui_arm32_tls_fix.md)
- **RUNS on OHOS ARM32 QEMU** — GoogleTest main() reached, clean exit
- Root cause: `dynlink.o` in musl `libc.a` overrides `__init_tls` and `__libc_start_init` for static binaries
- Fix: rename symbols in `dynlink.o`, provide custom `__init_tls` assembly
- Patched libc: `ohos-sysroot-arm32/usr/lib/libc_static_fixed.a`
- Custom TLS: `arkui_test_standalone/custom_init_tls.S`
- CMake toolchain: `/tmp/arm32-ohos-toolchain.cmake`
- Build: `cmake -DCMAKE_TOOLCHAIN_FILE=... && make button_test_ng`
- overlay_manager.cpp: patched lazy init for RefPtr<Curve> globals
- ARM32 virtual thunk offsets differ from x86_64 (n96/n128/n360 vs n192/n256/n720)
