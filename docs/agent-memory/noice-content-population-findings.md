## ROOT of empty display: UI gates on a network fetch the child can't do
LibraryFragment.onViewCreated → `LibraryViewModel.e()` triggers `loadLibraryManifestInCacheStore` (the manifest fetch). The list Flow appears gated on that load rather than serving the Room cache independently. The fetch CANNOT succeed:
- **Child has NO network: `getaddrinfo` returns EPERM** (`android.system.ErrnoException EPERM` → `GaiException EAI_SYSTEM` → uncaught `SecurityException: Permission denied (missing INTERNET permission?)` → FATAL crash in the `i8.s` Dispatcher coroutine). This is a real OHOS appspawn-x-child inet permission/namespace wall — board internet being ON does NOT help the child. Confirmed by removing libgaihook → noice crashes on the library fetch.
- With **libgaihook.so** (LD_PRELOAD via start_asx.sh) stubbing getaddrinfo to a dummy, noice doesn't crash but the fetch hangs/fails on connect → UI stays empty (cache not surfaced). gaihook is REQUIRED to prevent the crash.
- noice uses a **TlsShim** (not real Conscrypt; `[AESPROBE] prov[5]=TlsShim`) so even reachable HTTPS would likely fail the handshake.

## DEFINITIVE ROOT CAUSE of the empty list (2026-06-02 final): Android service binding fails
The displayed library list (`LibraryViewModel` field `m`, a transform/combine) mixes the library data with `SoundPlaybackService` state flows (bindServiceCallbackFlow: getState/getSoundStates/getSoundVolumes/getCurrentPreset). **The service NEVER connects** → those flows never settle → the combine never emits → adapter gets 0 rows. WHY it never connects (root): adapter `ActivityManagerAdapter.bindService` (oh-adapter-framework.jar) does `registerConnection` → `nativeConnectAbility(bundleName, abilityName, connId)` (OH `IAbilityManager.ConnectAbility`); for noice's `SoundPlaybackService` it returns **non-zero (fail)** → immediate `unregisterConnection` → `return 0` (bind failed); noice retries 9× (connId 1-9, ~0.7s) then gives up; `onServiceConnected` never fires. The adapter does NOT surface app-defined **Android services as connectable OH abilities** — a fundamental adapter capability gap (HelloWorld never exercised bindService; noice is the first). Source: `/home/[user]/scope-b-workdir/src/framework/activity/java/ActivityManagerAdapter.java` (bindService ~line 340), `ServiceConnectionRegistry.java`, `jni/ability_connection_adapter.cpp`. ### The actual final crash + fix (NOT a deep hwui GL wall — it was a Java NPE)
The "populated render crashes" was a **Java NPE during RecyclerView row measure**, captured via the child stderr stack (main thread, not RenderThread):
`java.lang.NullPointerException: write to field 'List FlexboxLayout.v' on a null object reference in FlexboxLayout.setFlexLines` at `FlexboxLayout.onMeasure`. The sound-row layout (ConstraintLayout) contains a `com.google.android.flexbox.FlexboxLayout` (tag chips); its flex-line computation writes field `v` (mFlexLines) and the receiver reads as null under this ART/R8 build → crashes every row measure (any non-empty list). DfxSignalHandler couldn't dump it because it's a caught-then-rethrown Java exception in invokeStaticMain, NOT a native signal — earlier "RenderThread/OnRemoteDied" was a misread; the real stack is in the child stderr (`adapter_child_<pid>.stderr` / hilog `[stderr]`/`J_invokeStaticMain_main_threw`).
**FIX:** replaced `FlexboxLayout.onMeasure(II)V` with a minimal safe measure (no flex computation, no `v`-write): `getDefaultSize(0,widthSpec)` for width, height 0, `setMeasuredDimension`. Tags chips collapse but the row (name/icon/star) measures → **list renders**. (smali in /tmp/noice_smali, deployed.)
