**The EGL wall (RENDERING #9) is ROOT-CAUSED and the core is FIXED (2026-06-01, this session).**

## Root cause (proven by G3.3 RequestBuffer probe in getOhNativeWindow)
The OH producer surface from `RSSurfaceNode::GetSurface()` is FUNCTIONAL (`RequestBuffer rc=0`, geom 720x1280, queue=3, consumer wired) — so the old "consumer-not-ready / timing race" theory was WRONG (that's why G2.9/G3.0/G3.1 flush attempts all failed). The real defect: the surface reports **`defFormat=0` (UNSET)** and **`defUsage=0x408` (NO HW_RENDER bit 0x100)**. `eglCreateWindowSurface` needs a valid pixel format + GPU-write usage, so it returned `EGL_NO_SURFACE` → libhwui abort. **Deterministic** (s014 was a rare alloc fluke, not timing).

## The fix (G3.4b — bridge, `oh_window_manager_client.cpp` getOhNativeWindow)
- `producerSurface->SetDefaultUsage(du | 0x100 | 0x200)` (+HW_RENDER +HW_TEXTURE) → verified `defUsage_lo32=0x708`, rc=0. (`SetDefaultFormat` on the producer returns NOT_SUPPORT 50102000 → format must be set on the NativeWindow instead.)
- After `CreateNativeWindowFromSurface`: `OH_NativeWindow_NativeWindowHandleOpt(nw, 3=SET_FORMAT, 12=RGBA8888)` + `(nw, 5=SET_USAGE, du|0x300)`; GET-readback confirms `GET_FORMAT=12 GET_USAGE=0x708`.
- **RESULT: the 1st `eglCreateWindowSurface` now SUCCEEDS** (hijack log: no error after 1st create). This was the wall that blocked the whole project (Netflix/Spotify/Maps too).

## Remaining blocker (being fixed: G2.8b SurfaceControl cache)
After the successful 1st create, a relayout triggers a surface RE-CREATE ~760ms later → 2nd `eglCreateWindowSurface` FAILS (`egl.eglCreateWindowSurface error`) → libhwui abort BEFORE frame #1 draws. Cause: `WindowSessionAdapter.java` relayout builds a **NEW `SurfaceControl` every call** (line ~657, no cache) → ViewRootImpl sees a changed surface each traversal → hwui destroys+recreates EGLSurface; the recreate fails (can't bind a 2nd window surface to the same OH producer). Also the deployed ohaf jar is STALE (lacks the source's `[G2.8]` reverse-push once-guard — device log shows old "resized:" not "(once)"). FIX: cache the SC per window binder (`mSurfaceControls`) + deploy current source (has G2.8 guard) via boot-regen.

## 3rd layer: multi-window SC session collision (2026-06-02) — root-caused, fix blocked on .so reproducibility
With G2.8b grafted, noice's create#1 succeeds but it has **TWO windows**: session 13 (AppIntroActivity, surfaceNode ...401) + session 12 (MainActivity, ...403). `getOhNativeWindow` ran ONLY for session 12 → only Main's surface got the format/usage fix; the other window's EGL create failed → abort. ROOT: `sc_to_oh_native_window` (android_graphics_compat_shim.cpp) uses the SC's attached session if present, ELSE falls back to `oh_wm_get_last_session()` (thread-local, last-wins). SCs are created with `sessionId=0` (`alloc_sc` in android_view_SurfaceControl.cpp) — `nativeAttachSessionToSc` JNI was REMOVED (G2.14r) — so BOTH windows fall back to the last-attached session → collide on one nw. FIX = stamp each SC's session at creation (`alloc_sc`: `sessionId = oh_wm_get_last_session()`, since relayout sets it right before building each window's SC; copyFrom already propagates sessionId). **BUT applying it requires rebuilding `liboh_android_runtime.so`, which is NON-REPRODUCIBLE from [user]: my rebuild (359040 vs deployed 389420) is MISSING the SQLite JNI** → noice dies `UnsatisfiedLinkError: SQLiteConnection.nativeOpen` in a Dispatchers.IO coroutine BEFORE window setup. HW-gate passed (HelloWorld doesn't use SQLite) so it didn't auto-catch. **ROLLED BACK the .so (restored ce319431 w/ SQLite).** Both `android_view_SurfaceControl.cpp` (alloc_sc/resolve) AND `android_graphics_compat_shim.cpp` (sc_to_oh_native_window) live in `liboh_android_runtime.so` → the stamp-side AND resolve-side fixes are both blocked by this .so wall. **REMAINING PATH (proven techniques, multi-component, NOT a .so rebuild):** re-add `nativeAttachSessionToSc(SurfaceControl, int)` JNI in the BRIDGE (`liboh_adapter_bridge.so`, reproducible) → extract `sc.mNativeObject` (GetLongField) → call the deployed .so's already-EXPORTED `oh_sc_attach_session(ptr, sessionId)`; AND call it from `WindowSessionAdapter.relayout` after build/copyFrom (Java rebuild → re-graft into dfb3a0c2 ohaf jar). ## REAL final crash (2026-06-02): SIGSEGV in ASurfaceControl_release (double-free) — NOT a reap
The "~8s reap" was a MISDIAGNOSIS. Faultlog (cppcrash-2367) shows: `SIGSEGV(SEGV_MAPERR)@0xe6a8b014, Tid:RenderThread, #00 ASurfaceControl_release+20 /system/lib/liboh_android_runtime.so`. After both EGL surfaces succeed (graphics works), hwui's RenderThread **double-releases** an ASurfaceControl: `ASurfaceControl_create` sets refcount=1; `ASurfaceControl_release` does `if(--refcount==0){destroy; delete sc;}` — the 2nd release derefs the freed struct → SIGSEGV (now reached because G3.6 makes both windows' surfaces succeed → 2-window RenderThread teardown). FIX (G3.7): don't `delete sc` (leak the 3-word struct; 2nd release sees old-refcount≠1 → harmless no-op; holder destroyed once). Source fixed in android_view_SurfaceControl.cpp, BUT liboh_android_runtime.so is NON-REPRODUCIBLE ([user] source has NO SQLite JNI; the on-device .so has it). So **BINARY-PATCHED the deployed .so**: ASurfaceControl_release `delete sc` is a tail-call `b 0x3ea00` at vaddr 0x23094 (file off 0x22094, bytes `59 6e 00 ea`) → replaced with `bx lr` (`1e ff 2f e1`) via python ELF-patcher → patched .so md5 **f82d8cdc** (vs deployed ce319431). Verified via llvm-objdump (0x23094: bx lr). Patch script pattern + /tmp/runtime_patch_deploy.sh.
