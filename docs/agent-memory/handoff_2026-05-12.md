## Architectural reminders (DON'T violate)

1. **Subtractive validation, not additive shimming** (`memory/feedback_subtraction_not_addition.md`)
   - Start from real-Android working baseline; subtract layers one at a time until break.
   - NEVER speculatively add a shim. Each new shim has to be justified by a concrete subtractive-harness failure.
   - `docs/engine/AGENT_SWARM_PLAYBOOK.md` §3 has the full anti-pattern list.

2. **No per-app hacks** (`memory/feedback_no_per_app_hacks.md`)
   - No `if (pkg.startsWith("com.github.ashutoshgngwr."))` style branches.
   - No `NoiceResourceMap`, `NoiceStringMap`, McD constants. C1-C5 cleaned these out — don't reintroduce.
   - When something doesn't work for app X, fix the Android API surface generically.

3. **Fail-loud, not silent no-op** (CR2 / CR17 / CR19 pattern)
   - Unobserved service methods throw `ServiceMethodMissing.fail("serviceName", "methodName")`, not return null/0/false.
   - This way, every new app surfaces real gaps instead of hiding them.

4. **Binder pivot, not framework-class shim** (`memory/project_binder_pivot.md`)
   - When a service method NPEs, implement it inside the relevant `WestlakeXxxService.Stub` subclass — not via reflection on framework internals.
   - `WestlakeFragmentLifecycle`-style bypasses are explicitly forbidden.
   - Same-process `Stub.asInterface` optimization is on via `nativeGetLocalService` — Java service calls are zero-cost.
