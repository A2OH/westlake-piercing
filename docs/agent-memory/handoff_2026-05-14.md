## Current blockers

1. **CR56 in flight — `WestlakeContextImpl.getApplicationContext()` returns
   null.** Single-method shim: return `mAttachedApplication` or fall back
   to `WestlakeActivityThread.currentApplication()`. M7-Step2 §4.3
   specifies the exact patch. This unblocks Hilt lazy delegate resolution
   in the production path. Anti-drift contract applies — additive on a
   class WE own.

2. **M8-Step2 in flight** — symmetric McD production launcher driving
   `Instrumentation.callActivityOnCreate` for `Hilt_SplashActivity`.
   Expect to expose the same `getApplicationContext` gap; CR56 fix may
   resolve both apps.

3. **M7-Step3 series of in-body NPEs** — once getApplicationContext is
   wired, the next likely NPEs are `Configuration.setTo(null)`,
   `getSharedPreferences` (currently returns null with deferral comment),
   and possibly `getCacheDir/getFilesDir`. Each is one-method-on-a-class-
   we-own work; expect 1-3 person-days of discovery + impl.

4. **Real Binder traffic into M4 services has not been exercised.** Once
   apps make actual `getService(...)` calls that fall through the
   local-elision optimization, transaction codes will surface and Tier-1
   service methods will need real bodies. CR2 + CR19 fail-loud pattern
   means we surface this loud, not silent.

5. **PF-arch-054 underlying substrate bug remains open** (CR26 sidestepped
   it via PF-arch-055 lambda rewrite, but `env->functions` pointer
   corruption root cause is still unknown). Not blocking; documented in
   PHASE_1_STATUS.md headline.

## Architectural reminders (DON'T violate)

1. **Macro-shim contract is now mandatory in every brief**
   (`feedback_macro_shim_contract.md`). Forbidden: `Unsafe.allocateInstance`
   on framework.jar classes, `Field.setAccessible` on framework internals,
   "planting" state on framework's ResourcesImpl/AssetManager/Configuration,
   per-app branches, adding new methods to `WestlakeContextImpl` (CR22
   freeze). Permitted: implementing API methods on classes WE own (V2
   shadows) with AOSP-default verbatim, safe-primitive, or delegation
   bodies. Self-audit gate runs before reporting complete.

2. **Subtractive validation, not additive shimming**
   (`feedback_subtraction_not_addition.md`). Start from a real-Android
   working baseline; subtract layers one at a time until break. NEVER
   speculatively add a shim.

3. **No per-app hacks** (`feedback_no_per_app_hacks.md`). No
   `if (pkg.equals("com.mcdonalds.app"))` branches. No `NoiceResourceMap`,
   `NoiceStringMap`, McD constants. Fix the Android API surface generically.

4. **Fail-loud, not silent no-op** (CR2 / CR17 / CR19 pattern). Unobserved
   service methods throw `ServiceMethodMissing.fail("serviceName",
   "methodName")`. CR19 converted 168 silent stubs in
   `WestlakePackageManagerStub` per codex review #2 Tier 2.

5. **Binder pivot, not framework-class shim**
   (`project_binder_pivot.md`). When a service method NPEs, implement
   it inside the relevant `WestlakeXxxService.Stub` subclass — not via
   reflection on framework internals.

6. **V2 substrate boundary is frozen** — V2-Step1..Step8-fix landed all
   the in-process Java shadow classes. Don't add new fields to those
   classes opportunistically; if a new fail comes from them, redesign
   per `BINDER_PIVOT_DESIGN_V2.md`.
