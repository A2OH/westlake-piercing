## Anti-pattern recognition (V3-specific; additions to 2026-05-16 list)

7. **Host exit-code laundering.** `hdc.exe` returns host exit 0 regardless
   of device exit. Boolean control-flow against raw `hdc shell` is
   permanently TRUE. Use `hdc_shell_check`. See
   [[feedback_hdc_shell_check_pattern]].

8. **`|| true` on a chcon, push, or chroot setup step.** Same family as
   anti-pattern #7. Prefer explicit allowlist of legitimately-optional
   files; everything else is fatal at pre-flight.

9. **Per-binary ldd walking for chroot deps.** Use RO bind-mount of host
   lib paths instead. See [[feedback_chroot_dynamic_elf_ro_bind]].

10. **/system writes before engine evidence.** Phase 2 (/system) is
    deferred until Phase 1b/1c lands chroot-confirmed engine evidence.
    Don't reintroduce brick-risk by skipping Phase 1.
